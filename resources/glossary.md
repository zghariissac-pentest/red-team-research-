# Glossary

## Overview

This glossary provides definitions for common cybersecurity, penetration testing, red teaming, networking, and security operations terms used throughout this repository.

---

# A

## Active Directory

A directory service developed by Microsoft that manages users, computers, groups, policies, and authentication within Windows environments.

## Adversary Emulation

The process of replicating the tactics, techniques, and procedures of real-world threat actors in a controlled environment.

## API

Application Programming Interface. A set of rules and mechanisms that allow software applications to communicate with one another.

## Attack Surface

The total number of possible entry points that an attacker can use to interact with a target system.

---

# B

## Blue Team

Security professionals responsible for defending systems, monitoring activity, detecting threats, and responding to incidents.

## Brute Force Attack

An attack method that attempts multiple credential combinations until a valid one is found.

## Buffer Overflow

A software vulnerability caused by writing more data into memory than allocated.

---

# C

## C2

Command and Control. Infrastructure used to communicate with compromised systems during security operations or adversary simulations.

## CVE

Common Vulnerabilities and Exposures. A publicly known identifier assigned to security vulnerabilities.

## CVSS

Common Vulnerability Scoring System. A framework used to measure vulnerability severity.

---

# D

## Defense in Depth

A security strategy that uses multiple layers of protection to reduce risk.

## DNS

Domain Name System. A protocol used to translate domain names into IP addresses.

---

# E

## Enumeration

The process of gathering detailed information from systems, services, and applications.

## Exploit

Code or a technique that takes advantage of a vulnerability to achieve unintended behavior.

---

# F

## Firewall

A security control that filters network traffic according to predefined rules.

## Fuzzing

A testing technique that sends unexpected or malformed inputs to identify software weaknesses.

---

# H

## Hash

A fixed-length representation of data generated using a hashing algorithm.

## Honeypot

A decoy system designed to attract and monitor malicious activity.

---

# I

## IDS

Intrusion Detection System. A system that monitors activity and generates alerts for suspicious behavior.

## IPS

Intrusion Prevention System. A system capable of detecting and blocking malicious activity.

## IOC

Indicator of Compromise. Evidence suggesting a system may have been compromised.

---

# K

## Kerberos

An authentication protocol commonly used in Active Directory environments.

---

# L

## Lateral Movement

The process of moving from one compromised system to another within an environment.

## Least Privilege

A security principle that grants only the permissions necessary to perform a task.

---

# M

## MITRE ATT&CK

A knowledge base documenting adversary tactics, techniques, and procedures observed in real-world attacks.

## Malware

Software designed to perform unauthorized or harmful actions on a system.

## MFA

Multi-Factor Authentication. An authentication mechanism requiring multiple forms of verification.

---

# N

## Nmap

A network scanning tool used for host discovery, port scanning, and service enumeration.

---

# O

## OSINT

Open Source Intelligence. Information collected from publicly available sources.

## OWASP

Open Worldwide Application Security Project. An organization focused on improving software security.

---

# P

## Pentest

Short for penetration test. An authorized security assessment designed to identify vulnerabilities.

## Privilege Escalation

The process of obtaining higher permissions than originally granted.

## Payload

Code or instructions delivered during exploitation to achieve a specific objective.

---

# R

## Reconnaissance

The process of gathering information about a target before testing or assessment activities.

## Red Team

Security professionals who simulate realistic adversary behavior to evaluate defenses.

## Reverse Engineering

The process of analyzing software to understand its functionality and behavior.

---

# S

## SIEM

Security Information and Event Management. A platform used to collect, analyze, and correlate security events.

## SOC

Security Operations Center. A team responsible for monitoring and responding to security incidents.

## SQL Injection

A vulnerability that allows unintended interaction with a database through application inputs.

---

# T

## Threat Intelligence

Information about threats, threat actors, capabilities, and attack techniques used to support security decisions.

## TTPs

Tactics, Techniques, and Procedures used by adversaries during operations.

---

# V

## Vulnerability

A weakness that could be exploited to compromise confidentiality, integrity, or availability.

## Vulnerability Assessment

The process of identifying, classifying, and prioritizing security weaknesses.

---

# W

## WAF

Web Application Firewall. A security solution designed to protect web applications from attacks.

## Web Application

Software that runs on a web server and is accessed through a browser.

---

# X

## XSS

Cross-Site Scripting. A vulnerability that allows malicious scripts to execute within a user's browser.

---

# Zero Trust

A security model based on continuous verification and the assumption that no user or device should be trusted by default.
