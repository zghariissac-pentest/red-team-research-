# technical-report
## Purpose

The technical report presents detailed findings for technical teams. Each finding includes an explanation, impact, and how to fix it.

## Structure
### Finding Title

Clear short name.

Description

Explain what the issue is and why it matters.

#### Impact

Explain what an attacker can do if they exploit it.

### Evidence

Show how it was discovered.
(Commands, screenshots, or logs can be added here in real assessments.)

Remediation

Explain how to fix the issue in a practical way.
