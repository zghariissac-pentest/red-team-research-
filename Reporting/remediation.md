# remediation
## Purpose

This document provides clear, actionable steps to fix the issues discovered during the assessment.

# Key remediation steps
## 1. identity and password hardening

Enforce MFA

Apply strict password policies

Disable unused accounts

## 2. AD configuration fixes

Correct weak ACLs

Remove dangerous delegation

Harden group memberships

## 3. host-level improvements

Patch outdated systems

Remove local admin reuse

Enable logging and monitoring

## 4. network security

Segment critical systems

Limit lateral movement paths

Reduce unnecessary services

## 5. monitoring and detection

Centralize logging

Alert on privilege changes

Monitor authentication anomalies
